**Part A: Add 2 internal mutations to `convex/kbInsights.ts`**

Add these AFTER the existing `setExtracting` mutation at the bottom of the file:

1. `setSummaryGenerating` — internal mutation mirroring `setExtracting` pattern but for the `summaryGenerating` flag:
   - Args: `{ generating: v.boolean() }`
   - If row exists: `ctx.db.patch(existing._id, { summaryGenerating: args.generating })`
   - If no row AND `generating=true`: insert minimal placeholder row with `{ extractedAt: 0, documentCount: 0, extracting: false, metrics: [], summaryGenerating: true }`
   - This is a SEPARATE flag from `extracting` — do NOT touch `extracting`

2. `saveSummary` — internal mutation that uses `ctx.db.patch()` (NOT delete-then-insert) to preserve existing metric data during regeneration (SUM-04 requirement):
   - Args: `{ summaryBullets: v.array(v.string()), summaryGeneratedAt: v.number() }`
   - If row exists: `ctx.db.patch(existing._id, { ...args, summaryGenerating: false })`
   - If no row (edge case — metrics never extracted): `ctx.db.insert("kbSummaryCache", { extractedAt: 0, documentCount: 0, extracting: false, metrics: [], ...args, summaryGenerating: false })`
   - CRITICAL: MUST use `ctx.db.patch()` on the existing row. Using `ctx.db.delete()` + `ctx.db.insert()` would erase the previous summaryBullets during regeneration, blanking the UI (violates SUM-04).

**Part B: Add `generateSummary` action to `convex/kbInsightsActions.ts`**

Add the `SUMMARY_SYSTEM_PROMPT` constant AFTER the existing `EXTRACTION_SYSTEM_PROMPT`:

```typescript
const SUMMARY_SYSTEM_PROMPT = `You are an executive briefing assistant for the Dads' Education Center (DEC), a nonprofit.

Generate exactly 3-5 concise bullet points summarizing organizational highlights from the provided documents. Each bullet should be boardroom-ready — factual, specific, and useful for stakeholder conversations.

RULES:
1. Use only information explicitly stated in the documents. Do not invent or estimate.
2. If KPI metrics are provided, reference them directly (e.g., "47 clients served in Q4") — do not contradict them.
3. Include both wins and areas needing attention for a balanced picture.

5. Return ONLY the bullet points, one per line, starting with "- ".
6. Write 3-5 bullets — no more, no fewer.`;
```

Add the `generateSummary` action AFTER the existing `extractMetrics` action. It follows the SAME pattern as `extractMetrics` but with these differences:
- Uses `setSummaryGenerating` (not `setExtracting`) for the generating flag
- Uses `saveSummary` (not `saveCache`) for persistence — patch-based, preserves existing data
- Does NOT use `response_format: { type: "json_schema" }` — plain text output is sufficient
- Uses `temperature: 0.3` (not `0`) — slightly creative for natural prose
- Injects extracted KPI metrics as additional context to avoid contradicting stat cards
- Parses bullets from raw text: split on `\n`, strip leading `- ` / `* ` / `• `, filter empty, `.slice(0, 5)` hard cap

Full implementation pattern:

```typescript
export const generateSummary = action({
  args: {},
  handler: async (ctx) => {
    // 1. Set generating flag (patch, not delete-insert)
    await ctx.runMutation(internal.kbInsights.setSummaryGenerating, { generating: true });

    try {
      const OpenAI = (await import("openai")).default;
      const apiKey = await getOpenAIApiKey(ctx);
      const openai = new OpenAI({ apiKey });

      // 2. Load all KB files (same as extractMetrics)
      const files = await ctx.runQuery(api.knowledgeBase.listFiles);
      if (files.length === 0) {
        await ctx.runMutation(internal.kbInsights.setSummaryGenerating, { generating: false });
        throw new Error("No KB documents uploaded. Upload documents in the Knowledge Base tab before generating a summary.");
      }

      // 3. Download text content (reuse isBinaryContent check)
      const documentBlocks: string[] = [];
      for (const file of files) {
        const url = await ctx.storage.getUrl(file.storageId);
        if (!url) continue;
        const response = await fetch(url);
        const text = await response.text();
        if (text.length > 0 && !isBinaryContent(text)) {
          documentBlocks.push(`=== Document: ${file.fileName} ===\n${text}`);
        }
      }

      if (documentBlocks.length === 0) {
        await ctx.runMutation(internal.kbInsights.setSummaryGenerating, { generating: false });
        throw new Error("No readable text documents found in KB.");
      }

      // 4. Inject extracted KPI metrics as reference context
      const cache = await ctx.runQuery(api.kbInsights.getCache);
      const kpiContext = cache?.metrics
        ?.filter((m: { value: string | null }) => m.value !== null)
        .map((m: { label: string; value: string | null; unit: string | null }) =>
          `${m.label}: ${m.value} ${m.unit ?? ""}`.trim()
        )
        .join(", ") ?? "";

      // 5. Chat Completions — plain text (no json_schema needed)
      const completion = await openai.chat.completions.create({
        model: "gpt-4o",
        temperature: 0.3,
        messages: [
          { role: "system", content: SUMMARY_SYSTEM_PROMPT },
          {
            role: "user",
            content: [
              kpiContext
                ? `Known KPI metrics (use as reference, do not contradict): ${kpiContext}`
                : "",
              `Summarize these ${documentBlocks.length} documents:\n\n${documentBlocks.join("\n\n")}`,
            ]
              .filter(Boolean)
              .join("\n\n"),
          },
        ],
      });

      const rawText = completion.choices[0].message.content ?? "";
      // Parse bullets: split on newlines, strip leading bullet markers
      const bullets = rawText
        .split("\n")
        .map((l) => l.replace(/^[-•*]\s*/, "").trim())
        .filter((l) => l.length > 0)
        .slice(0, 5); // hard cap at 5 bullets

      // 6. Persist using patch (preserves row, does not erase previous summary)
      await ctx.runMutation(internal.kbInsights.saveSummary, {
        summaryBullets: bullets,
        summaryGeneratedAt: Date.now(),
      });

      return { success: true, bulletCount: bullets.length };
    } catch (error) {
      await ctx.runMutation(internal.kbInsights.setSummaryGenerating, { generating: false });
      throw error;
    }
  },
});
```

Make sure to also strip any Markdown bold (`**...**`) from bullet text during parsing, since the frontend will handle its own styling. Add this to the map chain: `.replace(/\*\*(.*?)\*\*/g, "$1")`.
