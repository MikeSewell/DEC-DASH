<context>
## Current State

The dashboard has 11 sections managed via `SECTION_COMPONENTS` in `page.tsx` and `DEFAULT_DASHBOARD_SECTIONS` in `constants.ts`. Each section is wrapped in a `DashboardSection` card with collapse/reorder/hide controls.

**Existing components touched by this plan:**
- `ExecutiveSnapshot.tsx` — 3 stat cards (Cash on Hand, Revenue YTD, Total Expenses). StatCard inner component uses `text-2xl font-bold` for values.
- `AnalyticsCards.tsx` — 3 stat cards (Total Clients, Sessions, New Intakes). Its own StatCard uses `text-2xl font-bold`.
- `KBInsights.tsx` — MetricCard uses `text-2xl font-bold` for values.
- `ProfitLoss.tsx` — Shows 3 stat cards + doughnut chart + legend for expense breakdown. No progress bars currently.

**Old desktop app reference** (`dads-dashboard.css`):
- Thermometer: `.dads-thermometer` = 200px tall, rounded-20px, gradient fill from bottom, labels centered. `.dads-goal-percentage` = 2.5rem font-weight 900.
- Expense bars: `.dads-category-bar` = 8px height, border-radius 4px, `.dads-category-fill` = teal gradient, transition width 0.8s.
- Metric cards: `.dads-metric-value` = 2.75rem font-weight 800, hover translateY(-4px), 4px teal top-border on hover.

**Theme system**: `useTheme()` returns `{ resolvedTheme }`. Dark mode class `.dark` on `<html>`. All components use Tailwind theme tokens (text-foreground, bg-surface, border-border).

**Fallback pattern**: Components check for live data, fall back to constants from `dashboardFallbacks.ts`.

<task id="28-01-T1" title="Create FundingThermometer component with fallback data">
<description>
Create `src/components/dashboard/FundingThermometer.tsx` — a new dashboard section that renders a funding goal thermometer.

**Data source:**
- Use `useQuery(api.grants.getStats)` to get total awarded amount. If grants exist, the "current" value is totalAwarded and the "goal" is a configurable target (use $500,000 as default, stored as a const).
- If `getStats` returns no data or totalAwarded is 0, use `FALLBACK_FUNDING_GOAL` from dashboardFallbacks.ts.

**Add to `src/lib/dashboardFallbacks.ts`:**
```typescript
export const FALLBACK_FUNDING_GOAL = {
  current: 285400,
  goal: 500000,
  label: "FY 2024 Funding Goal",
} as const;
```

**Visual design (ported from old app `.dads-thermometer`):**
- Layout: flex row with thermometer bar on the left (flex-1) and stats on the right.
- Thermometer bar: 200px tall vertical bar with `rounded-2xl` corners. Background: `bg-border/30` (track). Fill: absolute-positioned div anchored to bottom, height set as percentage of current/goal, background is a vertical teal gradient (`from-primary-dark via-primary to-primary-light`). Fill animates via `transition-[height] duration-1000 ease-out`.
- Inside the thermometer, overlay the percentage label (large, bold, white text with text-shadow) centered vertically.
- Right side stats column:
  - Current amount: `text-3xl font-bold text-foreground` with `formatCurrency`.
  - Goal amount: `text-sm text-muted` with "Goal: $500,000".
  - Percentage: `text-4xl font-black text-primary` (the big hero number).
- Dark/light: Use theme tokens. Track bg in dark = `bg-border/20`, fill gradient uses `--primary` variables. The percentage number uses `text-primary`.
- Show fallback indicator (`"Sample data -- connect..."`) when using FALLBACK_FUNDING_GOAL.
- Component should use `useTheme()` for any non-token color decisions.
- Apply `hover-lift` to the outer container for consistency.
</description>
