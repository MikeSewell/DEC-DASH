  <name>Task 1: Add listAvailableCalendars action</name>
  <files>convex/googleCalendarActions.ts</files>
  <action>
Add a new public `action` called `listAvailableCalendars` to `convex/googleCalendarActions.ts`. This action:

1. Uses the same service account auth pattern as `syncCalendars` (import `googleapis`, create `GoogleAuth` with `GOOGLE_SERVICE_ACCOUNT_EMAIL` and `GOOGLE_PRIVATE_KEY` env vars, scope `https://www.googleapis.com/auth/calendar.readonly`).

2. Calls `calendar.calendarList.list()` to get all calendars the service account has access to. The Google Calendar API method `calendarList.list()` returns calendars the authenticated user (service account) has subscribed to or been granted access to.

3. Maps the response items to a clean array of `{ id: string, summary: string }` objects. Use `item.id` for the calendar ID and `item.summary` (falling back to `item.id`) for the display name. Filter out any items without an `id`.

4. Returns the array sorted alphabetically by summary.

5. If no calendars are found or the API call fails, return an empty array (do not throw — the UI will handle the empty state).

The action signature should be:
```typescript
export const listAvailableCalendars = action({
  handler: async () => {
    // ...
    return calendars; // Array<{ id: string; summary: string }>
  },
});
```

Do NOT modify the existing `syncCalendars` or `triggerSync` functions.
