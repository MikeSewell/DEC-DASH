**Component structure:**

import { useQuery, useAction } from "convex/react";

**State and data:**
- `const cache = useQuery(api.kbInsights.getCache);` — reactive subscription to cached metrics
- `const currentUser = useQuery(api.users.getCurrentUser);` — for role check
- `const extractAction = useAction(api.kbInsightsActions.extractMetrics);` — extraction trigger
- `const [extracting, setExtracting] = useState(false);` — local loading state
- `const [error, setError] = useState<string | null>(null);` — error display
- `const canExtract = currentUser?.role === "admin" || currentUser?.role === "manager";` — role gate

**handleExtract function:**
```typescript
const handleExtract = async () => {
  setExtracting(true);
  setError(null);
  try {
    await extractAction();
  } catch (err) {
    setError(err instanceof Error ? err.message : "Extraction failed");
  } finally {
    setExtracting(false);
  }
};
```

Also derive `isExtracting` from both local state AND the cache's `extracting` field:
```typescript
const isExtracting = extracting || (cache?.extracting ?? false);
```

**Rendering logic:**

1. **Loading state** (cache === undefined): Show a grid of 4 `StatCardSkeleton` components in a `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4` grid.

2. **Extract button area** — Render above the cards grid. Show:
   - If `canExtract`: An "Extract Metrics" button (or "Extracting..." when `isExtracting`). Style: `rounded-full border border-primary/30 bg-primary/5 px-4 py-1.5 text-sm font-medium text-primary hover:bg-primary/10 transition-colors disabled:opacity-50 disabled:cursor-not-allowed`. This follows the pill-shaped button convention from the design system.
   - "Last extracted" timestamp: If `cache?.extractedAt && cache.extractedAt > 0`, show `Last extracted ${timeAgo(cache.extractedAt)} from ${cache.documentCount} documents` in `text-xs text-muted`.
   - Error display: If `error`, show in `text-xs text-danger`.

3. **Empty state** (cache === null, or cache exists with extractedAt === 0 and no metrics):
   ```
   <div className="flex flex-col items-center justify-center py-12 text-muted">
     <svg ... sparkle/brain icon ... className="h-10 w-10 opacity-40 mb-3" />
     <p className="text-sm font-medium">No metrics extracted yet</p>
     <p className="text-xs mt-1">Upload documents in Knowledge Base, then click Extract Metrics</p>
   </div>
   ```

4. **Metric cards** — Filter metrics to only non-null values (`cache.metrics.filter(m => m.value !== null)`), then render each as a card. Per CONTEXT.md: "Dynamic card count — only show cards for metrics that were actually extracted (non-null)."

**Individual metric card design** — These must be visually DISTINCT from ExecutiveSnapshot's StatCards per the locked decision: "New 'AI Insights' card style visually distinct from QB-sourced StatCards — subtle AI badge or different accent color to signal document extraction origin."

Create an inline `MetricCard` component:
```typescript
function MetricCard({ metric }: { metric: { key: string; label: string; value: string | null; unit: string | null; sourceDocument: string | null; conflictValue?: string; conflictDocument?: string } }) {
  return (
    <div className="relative rounded-2xl border border-primary/20 bg-gradient-to-br from-surface to-primary/[0.03] p-5 shadow-[var(--warm-shadow-sm)] hover-lift">
      {/* AI badge — top-right corner */}
      <div className="absolute top-3 right-3 flex items-center gap-1 rounded-full bg-primary/10 px-2 py-0.5 text-[10px] font-semibold uppercase tracking-wider text-primary">
        <svg className="h-3 w-3" fill="none" viewBox="0 0 24 24" stroke="currentColor" strokeWidth={2}>
          <path strokeLinecap="round" strokeLinejoin="round" d="M9.813 15.904L9 18.75l-.813-2.846a4.5 4.5 0 00-3.09-3.09L2.25 12l2.846-.813a4.5 4.5 0 003.09-3.09L9 5.25l.813 2.846a4.5 4.5 0 003.09 3.09L15.75 12l-2.846.813a4.5 4.5 0 00-3.09 3.09z" />
        </svg>
        AI
      </div>

      {/* Metric value */}
      <div className="text-2xl font-bold text-foreground">
        {metric.value}
        {metric.unit && <span className="text-sm font-normal text-muted ml-1">{metric.unit}</span>}
      </div>

      {/* Metric label */}
      <p className="text-sm text-muted mt-1">{metric.label}</p>

      {/* Source document attribution */}
      {metric.sourceDocument && (
        <p className="text-xs text-muted/70 mt-2 truncate" title={metric.sourceDocument}>
          from {metric.sourceDocument}
        </p>
      )}

      {/* Conflict indicator — amber warning icon */}
      {metric.conflictValue && (
        <span
          title={`Conflict: ${metric.conflictDocument ?? "another document"} reported "${metric.conflictValue}"`}
          className="absolute bottom-3 right-3 text-warning cursor-help"
        >
          <svg className="h-4 w-4" fill="none" viewBox="0 0 24 24" stroke="currentColor" strokeWidth={2}>
            <path strokeLinecap="round" strokeLinejoin="round" d="M12 9v3.75m-9.303 3.376c-.866 1.5.217 3.374 1.948 3.374h14.71c1.73 0 2.813-1.874 1.948-3.374L13.949 3.378c-.866-1.5-3.032-1.5-3.898 0L2.697 16.126zM12 15.75h.007v.008H12v-.008z" />
          </svg>
        </span>
      )}
    </div>
  );
}
```

Key visual differences from StatCard:
- `border-primary/20` (tinted border) vs StatCard's `border-border`
- `bg-gradient-to-br from-surface to-primary/[0.03]` (subtle gradient) vs StatCard's flat `bg-surface`
- "AI" badge in top-right corner with sparkle icon
- No icon prop (StatCard has a large icon on the left)
- Source document attribution line

**Cards grid:**

**Responsive grid:** Per CONTEXT.md decision: "Responsive grid layout adapting to however many cards are present." The `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` handles 1-8 cards gracefully. If only 3 cards, the 4th column is empty on desktop — this is fine.

**Icon for each metric** — Rather than requiring individual icons per metric (increases complexity), the MetricCard uses the "AI" badge as its visual identifier. This is simpler and emphasizes the AI-extraction origin.
