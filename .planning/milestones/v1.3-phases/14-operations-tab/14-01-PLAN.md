Append three new exported queries to the BOTTOM of `convex/analytics.ts` (after the existing `getIntakeVolume` export). Do NOT modify any existing queries.

**1. `getAuditFeed` query (OPS-01):**
- No args needed (no authentication check -- the analytics page is already role-gated by the frontend ROLE_NAV_MAP).
- Query `auditLogs` using the `by_createdAt` index, ordered `desc`, and take the first 50 entries: `ctx.db.query("auditLogs").withIndex("by_createdAt").order("desc").take(50)`.
- For each log entry, resolve the user: `const user = await ctx.db.get(log.userId)`.
- Build a human-readable description string from the action + entityType + details:
  ```typescript
  function formatAction(action: string, entityType: string, details?: string): string {
    const verb = action.charAt(0).toUpperCase() + action.slice(1);
    const entity = entityType.replace(/_/g, " ");
    return details ? `${verb} ${entity}: ${details}` : `${verb} ${entity}`;
  }
  ```
  Place this helper inside the handler (not at module level) since it's only used here.
- Return: `Array<{ id: string; userName: string; userEmail: string; action: string; entityType: string; description: string; createdAt: number }>`.
- Map each log to: `{ id: log._id, userName: user?.name ?? "Unknown", userEmail: user?.email ?? "", action: log.action, entityType: log.entityType, description: formatAction(log.action, log.entityType, log.details), createdAt: log.createdAt }`.

**2. `getStaffActionStats` query (OPS-02):**
- No args.
- Collect ALL audit logs: `ctx.db.query("auditLogs").collect()`.
- Build a map of `userId -> { count: number; userId: Id<"users"> }`.
- After counting, resolve each unique userId to get name and email: `ctx.db.get(userId)`.
- Sort by count descending.
- Identify the most active user (first element after sorting, or null if no logs).
- Return:
  ```typescript
  {
    staffStats: Array<{ userId: string; userName: string; userEmail: string; actionCount: number }>;
    mostActive: { userId: string; userName: string; actionCount: number } | null;
    totalActions: number;
  }
  ```

**3. `getCategorizationStats` query (OPS-03 + OPS-04):**
- No args.
- Collect ALL expense allocations: `ctx.db.query("expenseAllocations").collect()`.
- Calculate acceptance rate:
  - `totalCategorized`: allocations where `status !== "error"` (i.e., pending + approved + submitted + skipped).
  - `accepted`: allocations where `status === "approved" || status === "submitted"`.
  - `acceptanceRate`: `totalCategorized === 0 ? 0 : Math.round((accepted / totalCategorized) * 100)`.
- Calculate category distribution:
  - Group by `finalClassName ?? suggestedClassName ?? "Uncategorized"`.
  - Build a map: `categoryName -> count`.
  - Convert to sorted array (by count desc): `Array<{ category: string; count: number }>`.
- Also compute confidence breakdown: counts for `high`, `medium`, `low` confidence.
- Return:
  ```typescript
  {
    totalAllocations: number;
    accepted: number;
    acceptanceRate: number;
    categoryDistribution: Array<{ category: string; count: number }>;
    confidenceBreakdown: { high: number; medium: number; low: number };
  }
  ```
