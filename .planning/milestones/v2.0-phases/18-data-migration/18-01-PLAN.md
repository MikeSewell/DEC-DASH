<task type="auto">
  <name>Task 1: Create convex/migration.ts with migrateAll internalMutation</name>
  <files>convex/migration.ts</files>
  <action>
Create `convex/migration.ts` with a single exported `migrateAll` internalMutation. This mutation combines enrollment creation and demographics backfill in one pass.

**Args:**
- `dryRun: v.optional(v.boolean())` — defaults to `false` when omitted

**Handler logic (in order):**

1. **Admin user lookup:** Query `users` table for the first user with `role === "admin"`. If none found, throw `new Error("No admin user found — cannot set createdBy on enrollments")`. Store as `adminUser`. This provides the required `v.id("users")` for `enrollments.createdBy`.

2. **Fetch all clients:** `await ctx.db.query("clients").collect()` — collect the full table (DEC has < 500 clients per research doc estimate).

3. **Per-client loop:** For each client:

   a. **Skip if no programId:** If `!client.programId`, increment `skipped` counter, push a warning string `"SKIP: ${client.firstName} ${client.lastName} — no programId"`, and `continue`.

   b. **Check for existing enrollment:** Query enrollments `withIndex("by_clientId", q => q.eq("clientId", client._id))` then `.collect()`. Check if any has `e.programId === client.programId` (the `alreadyEnrolled` flag). If already enrolled, increment `skipped` counter and skip enrollment creation (but still proceed to demographics backfill below).

   c. **Create enrollment (if not alreadyEnrolled):** Increment `enrollmentsCreated` counter. If `!dryRun`, insert into enrollments:
      - `clientId: client._id`
      - `programId: client.programId` (already checked non-null)
      - `status`: Map client status to enrollment status: `"active"` -> `"active"`, `"completed"` -> `"completed"`, `"withdrawn"` -> `"withdrawn"`. Default fallback for any unmapped value: `"active"`.
      - `enrollmentDate: client.enrollmentDate ?? client.createdAt` (fallback to createdAt when enrollmentDate missing)
      - `createdBy: adminUser._id`
      - `updatedAt: Date.now()`

   d. **Demographics backfill:** Fetch the linked intake forms:
      - `const legalIntake = await ctx.db.query("legalIntakeForms").withIndex("by_clientId", q => q.eq("clientId", client._id)).first()`
      - `const coparentIntake = await ctx.db.query("coparentIntakeForms").withIndex("by_clientId", q => q.eq("clientId", client._id)).first()`

      Build a `patch` object (`Record<string, string>`) with ONLY fields where the client value is currently falsy (null/undefined/empty string) AND the intake form has a value:
      - `referralSource`: from `legalIntake?.referralSource ?? coparentIntake?.referralSource`
      - `dateOfBirth`: from `legalIntake?.dateOfBirth ?? coparentIntake?.dateOfBirth`
      - `ethnicity`: from `legalIntake?.ethnicity ?? coparentIntake?.ethnicity`
      - `zipCode`: from `legalIntake?.zipCode ?? coparentIntake?.zipCode`
      - `email`: from `legalIntake?.email ?? coparentIntake?.email`
      - `phone`: from `coparentIntake?.phone` (legalIntakeForms does NOT have phone)

      If `Object.keys(patch).length > 0`:
        - Increment `demographicsUpdated` counter
        - If `!dryRun`: `await ctx.db.patch(client._id, patch)`

4. **Return report object:**
```typescript
return {
  mode: dryRun ? "DRY-RUN" : "EXECUTED",
  totalClients: clients.length,
  enrollmentsCreated,
  demographicsUpdated,
  skipped,
  warnings,
};
```

**CRITICAL constraints:**
- Use `internalMutation` (NOT `mutation`) — this must NOT be callable from the frontend.
- Do NOT call `enrollments.importBatch` — write directly to `ctx.db` to keep enrollment creation and demographics backfill in the same loop and avoid double-iteration.
- Do NOT overwrite already-populated client fields — only patch fields where the current value is falsy.
- The `warnings` array should be `string[]` collecting skip reasons for the dry-run report.
- Status mapping must handle all three client status values explicitly. Use a `const` map, not a ternary chain.
- Include a file header comment documenting the run commands: `npx convex run migration:migrateAll '{"dryRun":true}'` and `npx convex run migration:migrateAll`.
