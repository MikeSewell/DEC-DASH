<objective>
Ensure the Google Calendar cron sync pulls events only from the calendars that the admin has selected in the Admin > Google Calendar config page, not from all available calendars accessible to the service account. Also add cleanup of stale events from de-selected calendars.
</objective>

<context>
## Current State

The calendar sync already works correctly in terms of fetching from admin-selected calendars. Let me trace the flow:

1. **Cron entry (`convex/googleCalendarSync.ts`):**
   - `runSync` reads `getFullConfig` which returns the `googleCalendarConfig` singleton.
   - If no config or `config.calendars.length === 0`, it skips with a log message.
   - Otherwise calls `syncCalendars` action.

2. **Sync action (`convex/googleCalendarActions.ts`):**
   - `syncCalendars` reads `getFullConfig` again.
   - Iterates `config.calendars` (the admin-selected list) with `for (const { calendarId, displayName } of config.calendars)`.
   - For each selected calendar, fetches events from Google Calendar API using that `calendarId`.
   - Clears stale events for that calendarId, then upserts fresh events.

3. **Config schema (`convex/schema.ts`):**
   - `googleCalendarConfig.calendars` is an array of `{ calendarId, displayName }` — this IS the admin-selected list saved from the Admin UI.

**The issue from STATE.md:**
> "Cron sync update to use selected calendars from googleCalendarConfig (included in Phase 29 as INFRA-01)"

Looking at the code, `syncCalendars` already iterates `config.calendars` which ARE the selected calendars. However, there is a gap: when an admin de-selects a calendar (removes it from the config), the stale events from that calendar remain in `googleCalendarCache` forever. The sync only clears events for calendars that ARE in the config — it does not clean up events from removed calendars.

**Fix needed:** After syncing selected calendars, clean up any cached events whose `calendarId` is NOT in the current config.calendars list. This ensures de-selected calendars don't leave stale events in the cache.
</context>

const selectedCalendarIds = new Set(config.calendars.map(c => c.calendarId));
await ctx.runMutation(internal.googleCalendarInternal.cleanupDeselectedCalendars, {
  selectedCalendarIds: [...selectedCalendarIds],

**`convex/googleCalendarInternal.ts` — add `cleanupDeselectedCalendars` mutation:**

Add a new internal mutation:

```typescript
export const cleanupDeselectedCalendars = internalMutation({
  args: { selectedCalendarIds: v.array(v.string()) },
  handler: async (ctx, args) => {
    const selectedSet = new Set(args.selectedCalendarIds);
    // Fetch all cached events and remove those whose calendarId is not in the selected set
    const allEvents = await ctx.db.query("googleCalendarCache").collect();
    let removed = 0;
    for (const event of allEvents) {
      if (!selectedSet.has(event.calendarId)) {
        await ctx.db.delete(event._id);
        removed++;
      }
    }
    if (removed > 0) {
      console.log(`Cleaned up ${removed} stale events from de-selected calendars`);
    }
  },
});
```

This ensures:
- Events from calendars the admin has removed from their selection get cleaned up on every sync cycle.
- The cleanup runs AFTER successful sync of selected calendars, so it won't accidentally remove events if the sync partially fails.
- The log message helps with debugging.

**Note:** The `cleanupDeselectedCalendars` mutation iterates all cached events. With max ~100 events per calendar and typical 2-5 calendars, this is a small dataset (< 500 rows) — no performance concern.
