<task id="29-01-T2" title="Add gradient top-border hover accent to cards">
<description>
Add a teal gradient top-border that fades in on hover, matching the old desktop app's `.dads-metric-value` hover accent pattern (4px teal top-border on hover).

**`src/app/globals.css`:**
Update the `.hover-lift` CSS to include a `::before` pseudo-element that renders a gradient top border on hover.

Replace the existing `.hover-lift` block:
```css
/* -- Hover lift -- */
.hover-lift {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.hover-lift:hover {
  transform: translateY(-2px);
  box-shadow: var(--warm-shadow-md);
}
```

With:
```css
/* -- Hover lift -- */
.hover-lift {
  position: relative;
  overflow: hidden;
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.hover-lift::before {
  content: "";
  position: absolute;
  top: 0;
  left: 0;
  right: 0;
  height: 3px;
  background: linear-gradient(90deg, var(--primary), var(--accent));
  opacity: 0;
  transition: opacity 0.2s ease;
  z-index: 1;
  border-radius: 9999px 9999px 0 0;
}
.hover-lift:hover {
  transform: translateY(-2px);
  box-shadow: var(--warm-shadow-md);
}
.hover-lift:hover::before {
  opacity: 1;
}
```

This approach:
- Uses `position: relative` and `overflow: hidden` so the pseudo-element clips to the card's rounded corners.
- The gradient goes from `--primary` (teal #1B5E6B) to `--accent` (green #6BBF59) — matching DEC brand.
- The bar is 3px tall, at the very top of the card, with pill-shaped top corners.
- Fades in on hover via opacity transition (matches the 0.2s timing of the existing hover-lift).
- Works on ALL elements with `.hover-lift` — stat cards, DashboardSection cards, donation source cards, etc.
- No JS changes needed — purely CSS.

**IMPORTANT:** Check that DashboardSection's Card wrapper doesn't conflict. The Card component already has `overflow: hidden` via its children, but the `hover-lift` class is on the Card div itself, which has `rounded-2xl`. The `overflow: hidden` on `.hover-lift` will clip the pseudo-element to the card's border-radius. Verify that DashboardSection uses `className="overflow-hidden"` on the Card — it does (line 32), so the Card wrapper already allows this.
</description>
</task>
