**Current behavior:**
- `ExecutiveSnapshot`: When `qbConfig === null || qbConfig.isExpired`, renders an SVG icon + "Connect QuickBooks" message. DATA-01 requires it to instead show stat cards with fallback values.
- `ProfitLoss`: When `plResult === null`, renders a "No P&L data available" message. DATA-05 notes a $NaN bug — likely from `formatCurrency(undefined)` when `data` fields are undefined or from `parsePnlTotals` returning NaN. DATA-05 requires valid numbers plus a fallback expense breakdown.

**Fallback data guidelines (per STATE.md):**
- Hardcoded only — no Convex seed data
- Values must be plausible for a small US nonprofit (DEC is a nonprofit serving fathers/families in the legal/co-parenting space)
- Realistic FY 2024 nonprofit financials: total revenue ~$285k, expenses ~$248k, net income ~$37k
- Cash on hand: ~$54k (realistic nonprofit cash position)
- Expense categories: Program Services, Salaries & Benefits, Occupancy, Professional Fees, Marketing, Office Expenses

**$NaN root cause:** `parsePnlTotals` in `convex/quickbooks.ts` uses `parseFloat("0")` fallbacks so it shouldn't produce NaN on its own. The bug is more likely in `ProfitLoss.tsx` when `data` is non-null but a field value is `undefined` (e.g., `totalRevenue` undefined from destructuring), or when `expensesByCategory` is `{}` (empty object). The fix is two-pronged: (1) add `?? 0` guards on all `formatCurrency()` calls in `ProfitLoss.tsx`, and (2) use the fallback data when `plResult === null` or `!data`.
